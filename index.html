Git vs GitHub

الفرق الجوهري بين الأداة والمنصة — فهم عميق للمفاهيم والأوامر والاستخدامات الصحيحة لكل منهما

01

التعريف الجوهري

Git
Version Control System · أداة محلية

Git هي أداة سطر أوامر (CLI) تُنصَّب على جهازك المحلي. وظيفتها الوحيدة هي تتبع تغييرات الكود عبر الزمن — تحفظ لقطات (snapshots) من مشروعك في كل مرحلة، مما يتيح لك العودة لأي نسخة سابقة، إنشاء فروع مستقلة للتجربة، ودمج التعديلات من عدة مطورين.

التشبيه: Git كمحرك السيارة — يعمل على جهازك دون أن يعلم به أحد. لا يحتاج إنترنت، ولا خوادم خارجية.
النوع أداة CLI مفتوحة المصدر
يعمل بدون إنترنت ✓ نعم
يُنصَّب على جهازك المحلي
صاحبه Linus Torvalds · 2005
البديل SVN · Mercurial · Perforce
GitHub
Cloud Hosting Platform · منصة سحابية

GitHub هي منصة ويب تستضيف مستودعات Git على السحابة. تُضيف فوقها طبقة اجتماعية وتعاونية: Pull Requests للمراجعة، Issues لتتبع المشكلات، Actions للـ CI/CD، وصفحات لنشر المواقع. هي تستخدم Git خلف الكواليس لكنها ليست Git.

التشبيه: GitHub كمنصة YouTube — كما أن YouTube يستضيف فيديوهاتك ويتيح مشاركتها، GitHub يستضيف كودك ويتيح التعاون فيه.
النوع منصة SaaS (مملوكة لـ Microsoft)
يعمل بدون إنترنت ✗ لا
تعمل عبر المتصفح + git CLI
أُسست 2008 · اشترتها Microsoft 2018
البديل GitLab · Bitbucket · Gitea
02

مقارنة تفصيلية

ملاحظة أساسية
يمكنك استخدام Git دون GitHub تماماً — تتبع كودك محلياً دون رفعه لأي مكان. لكن لا يمكنك استخدام GitHub دون Git — المنصة تعتمد كلياً على Git كمحرك خلفي. الخلط بينهما خطأ شائع يقع فيه كثير من المبتدئين.
المحور Git GitHub
الماهية نظام التحكم بالإصدار VCS — الأداة ذاتها منصة استضافة تستخدم Git كمحرك
مكان التشغيل جهازك المحلي local machine خوادم سحابية cloud servers
الاتصال بالإنترنت غير مطلوب للعمليات المحلية مطلوب دائماً للوصول والتزامن
واجهة المستخدم سطر الأوامر CLI بشكل أساسي واجهة ويب + سطر أوامر معاً
التعاون ممكن لكن معقد بدون منصة مصمم خصيصاً للتعاون — PR + Review
الأتمتة يحتاج أدوات خارجية للـ CI/CD GitHub Actions مدمج بالكامل
إدارة المشاريع لا توجد — مجرد VCS Issues، Projects، Milestones، Wiki
نشر المواقع لا توجد GitHub Pages مجاناً للمستودعات العامة
الترخيص مفتوح المصدر بالكامل GPL-2.0 مملوك + خطة مجانية محدودة
البدائل SVN · Mercurial · Perforce GitLab · Bitbucket · Azure DevOps
03

المصطلحات الأساسية

Repository
مستودع · ريبو
مجلد المشروع بكامل تاريخه. يحوي كل الملفات وكل التعديلات منذ بداية المشروع.
git init / git clone
Commit
نقطة حفظ · لقطة
تسجيل رسمي لتغييرات معينة مع رسالة وصفية وختم زمني وهوية المؤلف.
git commit -m "..."
Branch
فرع · مسار مستقل
نسخة مستقلة من المشروع تُتيح العمل على ميزة دون التأثير على الكود الرئيسي.
git checkout -b feature
Merge
دمج · إدماج
دمج تغييرات فرع في فرع آخر. قد ينتج عنه تعارضات تحتاج حلاً يدوياً.
git merge feature-branch
Pull Request
طلب دمج · خاص بـ GitHub
طلب رسمي لدمج كودك في الفرع الرئيسي. يتضمن مراجعة من زملائك والموافقة.
GitHub UI → New PR
Fork
نسخة مستقلة · تفرع
نسخ مستودع شخص آخر لحسابك الخاص للعمل عليه بحرية دون التأثير على الأصل.
GitHub UI → Fork
Clone
استنساخ محلي
تحميل نسخة كاملة من مستودع بعيد (GitHub) لجهازك المحلي بكل تاريخه.
git clone [url]
Push / Pull
رفع / سحب
Push: رفع commits المحلية للمستودع البعيد. Pull: تحديث النسخة المحلية بآخر التغييرات.
git push origin main
Staging Area
منطقة التحضير
منطقة وسطى بين التعديل والـ commit. تختار فيها أي الملفات ستُضمَّن في الـ commit القادم.
git add .

05

أوامر Git الأساسية

// Setup & Init
  • git initتهيئة مستودع جديد في المجلد
  • git clone [url]استنساخ مستودع بعيد محلياً
  • git config --global user.nameضبط اسم المستخدم عالمياً
  • git statusعرض حالة الملفات الحالية
  • git log --onelineعرض تاريخ الـ commits مختصراً
// Remote & Sync
  • git remote add originربط المستودع بـ GitHub
  • git push origin mainرفع التغييرات لـ GitHub
  • git pull origin mainسحب آخر التغييرات من GitHub
  • git fetch --allجلب البيانات بدون دمج
  • git remote -vعرض الـ remotes المضافة
// Branching & Merging
  • git branchعرض كل الفروع
  • git checkout -b [name]إنشاء فرع جديد والانتقال إليه
  • git merge [branch]دمج فرع في الفرع الحالي
  • git rebase mainإعادة بناء الـ commits على main
  • git branch -d [name]حذف فرع بعد دمجه
// Saving & Undoing
  • git add .تحضير كل الملفات للـ commit
  • git commit -m "[msg]"تسجيل التغييرات مع رسالة
  • git stashحفظ التغييرات مؤقتاً بدون commit
  • git reset --soft HEAD~1التراجع عن آخر commit
  • git revert [hash]عكس commit بشكل آمن
06

متى تستخدم كلاً منهما؟

أريد حفظ نقاط تقدم في مشروعي الشخصي
Git فقط يكفيك. git init ثم git commit بانتظام. لا حاجة لرفع أي شيء.
أريد التعاون مع فريق في نفس الكود
Git + GitHub معاً. Git للعمل المحلي، GitHub للمشاركة والـ Pull Requests والمراجعة.
أريد نشر موقعي مجاناً
GitHub Pages خيار ممتاز. ارفع الكود لـ GitHub وفعّل Pages — الموقع يعمل على [user].github.io مجاناً.
أعمل في بيئة بدون إنترنت
Git وحده يعمل كاملاً offline. كل العمليات المحلية (commit، branch، merge) لا تحتاج اتصالاً.
أريد أتمتة الاختبارات والنشر
GitHub Actions يتيح CI/CD مدمجاً. كل push يُشغّل اختبارات تلقائية ويمكن نشر التطبيق فوراً.
هل أحتاج تعلم كليهما؟
نعم، لكن بترتيب. تعلم Git أولاً وأتقن الأوامر الأساسية. ثم انتقل لـ GitHub. بدون فهم Git ستظل محيراً في GitHub.
